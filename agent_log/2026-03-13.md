# 2026-03-13

- Added a Feishu-side acknowledgement reaction before provider execution starts, so valid Feishu requests now try to mark the source message with a lightweight bot-visible ack.
- Used Feishu's built-in `GLANCE` reaction type as the closest available equivalent to the Discord `👀` acknowledgement.
- Kept the reaction path fail-open: if the Feishu reaction API rejects the call, the provider turn still runs and the failure is only logged.
- Added a regression test for the Feishu reaction payload builder.
- Hardened `src/config.test.js` so the config suite no longer gets polluted by a developer's local `.env` credentials when `dotenv` runs during test startup.
- Updated `README.md` to document the new Feishu acknowledgement behavior.
