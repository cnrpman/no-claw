## 13:58 CST

- Switched Discord Gateway proxying away from the `ws` package path
- Findings:
  - local verification showed `ws` still timed out during the Discord Gateway opening handshake even when REST and plain HTTPS over the same proxy succeeded
  - Node's built-in `globalThis.WebSocket` succeeded against the same proxied Discord Gateway URL when `NODE_USE_ENV_PROXY=1`
- New behavior:
  - startup is now split into a small bootstrap file plus the main app module
  - the bootstrap applies proxy settings first, then patches `@discordjs/util.shouldUseGlobalFetchAndWebSocket()` before `discord.js` loads
  - this forces Discord Gateway traffic onto Node's built-in WebSocket implementation instead of `ws`
- Also reduced proxy warning noise by preferring `undici.ProxyAgent` when HTTP and HTTPS use the same upstream proxy
