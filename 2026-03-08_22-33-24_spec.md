# Discord Codex MVP Spec

生成时间: 2026-03-08 22:33:24 CST

## 目标

做一个最小可用的 Discord + Codex bridge。

用户体验目标:

- 在普通 channel 里，用户 `@codex` 提一个问题。
- bot 只读取这条用户命令本身，不读取整个 channel 历史，也不读取其他 thread。
- bot 以这条消息为起点创建一个新的 thread。
- bot 在该 thread 里回复用户。
- 之后用户在这个 thread 里继续 `@codex`，bot 继续在同一个 thread 里进行一问一答。

## 范围

当前阶段只做最小闭环:

- Discord mention 触发
- 普通 channel 自动创建 thread
- thread 内继续对话
- Codex session / thread resume 的基础接入

当前阶段不做:

- token 预算优化
- `compact` / 摘要压缩
- 自动读取整个频道历史
- 跨 thread 记忆
- 复杂工具调用编排

## 交互规范

### 1. 普通 Channel 首次提问

触发条件:

- 用户在普通 channel 发消息并 `@codex`

行为:

- bot 提取该条消息文本作为本次请求输入
- bot 创建一个新的 Discord thread
- bot 在该 thread 内发送回复

上下文规则:

- 只使用当前这条用户命令
- 不读取该 channel 的最近消息
- 不读取任何 thread 内容

### 2. Thread 内继续提问

触发条件:

- 用户在 bot 创建的 thread 内再次 `@codex`

行为:

- bot 将该 thread 识别为一个持续会话
- bot 通过 `codex` 的 thread / thread resume 机制继续当前对话
- bot 继续在同一个 thread 内回复

上下文规则:

- 默认基于该 Codex 会话已有上下文继续
- 不额外拼接父 channel 历史

### 3. 非触发消息

以下情况默认不触发:

- 没有 `@codex` 的普通消息
- 没有 `@codex` 的 thread 消息
- 与 bot 无关的其他频道消息

## 会话模型

核心映射关系:

- 一个 Discord thread 对应一个 Codex 会话

建议维护的数据:

- `discord_thread_id`
- `discord_parent_channel_id`
- `starter_message_id`
- `codex_thread_id` 或等价 resume 标识
- `created_by_user_id`
- `created_at`
- `last_activity_at`

工作方式:

- 首次 `@codex` 时创建新的 Codex 会话
- 将会话标识保存到本地存储
- 后续同一 Discord thread 内的 `@codex` 请求，使用保存的会话标识 resume

## 回复策略

bot 回复统一发在 thread 内。

普通 channel 中可选行为:

- 可发送一条很短的提示消息，告知“已在 thread 中回复”
- 或直接只创建 thread，不在主 channel 再额外发正文

MVP 建议:

- 主 channel 尽量少说话
- 正文全部放到 thread 里

## 错误处理

必须显式失败，不允许静默失败。

至少覆盖:

- thread 创建失败
- Codex 调用失败
- Codex resume 失败
- 本地会话映射缺失或损坏

失败时 bot 应在对应位置返回可操作信息，例如:

- 创建 thread 失败，请检查 bot 权限
- 会话恢复失败，请重新在主 channel `@codex` 发起新问题

## 可观测性

当前阶段只要求基础日志:

- 收到 mention
- 创建 thread
- 创建 Codex 会话
- resume 旧会话
- Codex 请求成功 / 失败

当前阶段不要求:

- token 精细统计
- `/status` 套餐用量聚合展示

## 后续实现建议

建议先做以下最小模块:

1. Discord event listener
2. Mention parser
3. Thread manager
4. Codex client wrapper
5. Session store
6. Basic logger
